You can use second tab of TeklaPad to create console based applications.

 

 

 

 

 

 

 

 

 

 

1- Each console application must have "Main" method (and It MUST be called "Main"):

 

static void Main()

 { 

    [your code]

 }

 

When the computer is asked to run a C# program, it needs to know where to start. It has been taught that, for C# programs, the programmer will always write a method called "Main" and that's where it must start - that's the entry point.

 

2- You must put your methods in a "class" block. So, the general structure of your code should be something like this:

 

using [your namespaces (Tekla.Structures.Model or System or...)]

class [any name (without space)]

    {

       static void Main()

        {

           [your code]

        }

    }

 

3- Finally, It's highly recommended to wrap your code in "try-catch" block to avoid run-time errors:

 

       static void Main()

        {

           try

             {

                [your code]

             }

           catch (Exception)

            {

               [exception handling (or you can leave it empty)]

            }

        }

 

Important:

All created *.exe files will be removed automatically after closing the TeklaPad.

So, you should move/copy your final program with its dependencies (Tekla.Structures.dll or...) to another location (for later usages) before closing the TeklaPad.
