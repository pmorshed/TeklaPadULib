Using TeklaPad in Drawing space. (Available since v 1.0.9)

 

1- Make sure that your drawing is open.

2- You can use DrawingDatabase or EntireDrawing variables to access selected or all  drawing objects.

3- Drawing object types are stored in predefined TSD (Tekla.Structures.Drawing) namespace alias in order to avoid conflicting with model object type names.

4- Keep in mind that "drawing object" is not the same as "model object". Drawings are just presentations of the model objects. Here is a quick comparison:

 

 

As another example, "Part" object type is not the same as "TSD.Part" type.

 

5- You can use GetModelObjectsOfType<T> to get collection of unique model objects (of type "T") from selected drawing objects (in DrawingDatabase or EntireDrawing).

 

Note: You can use all your model queries on drawings as well.

Just replace "Database.OfType<T>" with "DrawingDatabase.GetModelObjectsOfType<T>"  in your model query to create an equivalent drawing query.
