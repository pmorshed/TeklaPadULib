You can use second tab of TeklaPad to create plug-ins with "INP" or "Windows Form" UI.

 

Model plug-ins have to be inherited from PluginBase class and drawing plug-ins have to be inherited from the DrawingPluginBase calss.

 

 

How to create?

1- Choose a .NET Framework. Then, common assemblies will be added automatically.

 

 

2- Add needed system or/and Tekla API assemblies.

 

 

3- Choose "DLL"option from drop down list for your project.

 

 

4- Write your plug-in (you can also plug-in templates) and run the project. If everything goes fine, the created *dll file will be placed on output folder.

 

 

 

How to use?

1- Make sure that the Tekla Structures is closed.

3- Rename the created *.dll file (optional).

4- Put your *.dll file in ...\TeklaStructures\[version]\nt\bin\plugins folder. This adds your plug-in directly to the "Applications & components" catalog in Tekla Structures (you can also create a sub-folder inside "plugins" folder for your plugins)

5- Run (or restart) the Tekla Structures and look for the plug-in in the "Applications & components" catalog.

 

Hint: You can also add a thumbnail for your plug-in. Learn more here 

 

Important:

All created *.dll files will be removed automatically after closing the TeklaPad.

So, you should move/copy your final plug-in to another location (for later usages) before closing the TeklaPad

 

 

Key concepts of "INP" UI:

The user interface, or dialog, of a Tekla Open API plugin can be defined using the same definition language as Tekla Structures custom components and system components: input file format (INP).

 

Hint-1: Needed assemblies:

-> Tekla.Structures.Plugins.dll (Location: ~\[version]\nt\bin\plugins)

 

Hint-2: Needed namespaces:

using Tekla.Structures.Plugins;

 

Hint-3: General rules:

 

 

\n means: New line

\" (or "") means: " 

\"\" (or """") means: ""

 

 

Key concepts of "Windows Form" UI:

PluginFormBase class provides connectivity to Tekla Structures for plug-ins.

 

Hint-1: Needed assemblies:

-> Tekla.Structures.Plugins.dll (Location: ~\[version]\nt\bin\plugins)

-> Tekla.Structures.Dialog.dll  (Location: ~\[version]\nt\bin\dialogs)

 

Hint-2: Needed namespaces:

using Tekla.Structures.Plugins;

using Tekla.Structures.Dialog;

using System.Windows.Forms;
